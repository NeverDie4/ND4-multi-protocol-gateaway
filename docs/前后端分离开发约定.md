# 前后端分离开发约定

本项目当前按前后端分离模式开发：Go 后端作为 API Runtime，`mount-hub` 作为独立 Next.js 前端应用。

## 开发模式

| 模块 | 目录 | 默认地址 | 职责 |
|------|------|----------|------|
| 后端 | 项目根目录 | `http://127.0.0.1:5244` | 提供认证、文件系统、管理后台等 API |
| 前端 | `mount-hub/` | `http://127.0.0.1:3000` | 提供文件工作台和管理中心 UI |

前端通过环境变量访问后端：

```powershell
$env:NEXT_PUBLIC_API_BASE_URL = "http://127.0.0.1:5244"
npm.cmd run dev
```

后端本地启动命令：

```powershell
go run . server
```

后端会在 `data/` 下生成本地配置、SQLite 数据库和日志。`data/` 属于本地运行目录，不提交到 Git。

## public/dist 定位

`public/dist` 不是当前开发阶段的正式前端产物。

Go 后端源码中存在 `//go:embed all:dist`，因此本地编译后端时需要 `public/dist` 目录存在。当前 `public/dist` 仅用于满足后端编译和 API 联调启动要求，不承载 `mount-hub` 的正式构建结果。

开发阶段不要把 `mount-hub` 的 `.next`、`node_modules` 或临时日志复制进 `public/dist`。

## 本地联调产物

以下目录和文件属于本地运行产物，应保持未提交状态：

```text
data/
mount-hub/.next/
mount-hub/node_modules/
mount-hub/next-dev*.log
```

这些路径已在 `.gitignore` 中忽略。

## 发布路线

后续发布可选择两种路线之一：

1. 前后端分离部署：后端只提供 API，前端作为独立 Next.js 应用部署。
2. 后端内置前端资源：在明确需要单二进制发布时，再评估将前端静态导出并放入 `public/dist`。

在 UI 和 API 联调稳定前，默认采用前后端分离部署思路推进开发。
