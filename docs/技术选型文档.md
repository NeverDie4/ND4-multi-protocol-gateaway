## 二、数据库

### 2.1 支持的数据库引擎

| 引擎 | 使用场景 | 连接配置 |
|------|----------|----------|
| **MySQL** | 生产环境多实例 / 默认推荐 | `host / port / user / password / name` |
| **PostgreSQL** | 企业环境 / JSON 高级查询 | `host / port / user / password / name / ssl_mode` |

> SQLite3 在前期开发中可用 (`db_file` 模式)，但不纳入最终交付选型。

### 2.2 表结构 (GORM AutoMigrate, 13 张表)

| 表名 | Model | 用途 |
|------|-------|------|
| `x_storages` | Storage | 存储挂载配置（驱动、路径、缓存、代理策略） |
| `x_users` | User | 用户（密码哈希、角色、权限位掩码、2FA、SSO ID） |
| `x_roles` | Role | 角色（路径级权限作用域，JSON 序列化存储） |
| `x_metas` | Meta | 路径元数据（密码保护、隐藏规则、README、公告） |
| `x_setting_items` | SettingItem | 全局 KV 配置（含公开/只读/废弃标记） |
| `x_search_nodes` | SearchNode | Bleve 全文搜索索引节点 |
| `x_task_items` | TaskItem | 异步任务持久化（下载/复制/上传/解压） |
| `x_ssh_public_keys` | SSHPublicKey | SFTP 公钥认证 |
| `x_labels` | Label | 文件标签 |
| `x_label_file_bindings` | LabelFileBinding | 标签-文件绑定 |
| `x_obj_files` | ObjFile | 文件对象缓存 |
| `x_sessions` | Session | 用户会话（设备 IP / UserAgent / 过期时间） |
| `x_shares` | Share | 文件分享（密码/阅后即焚/访问限制/过期） |

> 表名前缀 `x_` 可通过 `table_prefix` 配置修改；MySQL 自动设置 `ENGINE=InnoDB CHARSET=utf8mb4`。

### 2.3 特殊字段设计

| 字段 | 存储方式 | 查询方法 |
|------|----------|----------|
| `Role.PermissionScopes` | `BeforeSave` Hook 将 `[]PermissionEntry` 序列化为 `raw_permission TEXT` | `AfterFind` Hook 自动反序列化 |
| `User.Role` | JSON 数组 `[1, 2, 3]` | MySQL `JSON_CONTAINS(role, '[1]')` |
| `User.Permission` | int32 位掩码（17 个权限位） | 位运算 `(perm >> bit) & 1` |
| `Storage.Addition` | TEXT 字段，存储驱动特有配置 JSON | 通过 `json.Unmarshal` 反序列化为对应驱动 struct |

### 3.1 核心选型

| 类别 | 选型 | 版本 | 用途 |

| 框架 | React | 19 | 函数组件 + Hooks |

### 3.2 组件库与样式

| 类别 | 选型 | 说明 |
|------|------|------|
| 组件库 | **shadcn/ui** | 基于 Radix Primitives + Tailwind CSS 4，**代码直接复制到本地**（非 npm 包），可自由修改源码 |
| CSS 方案 | Tailwind CSS 4 | 原子化 CSS，零运行时 |
| 图标 | **lucide-react** | shadcn/ui 官方配套图标库 |
| 主题 | CSS Variables + `next-themes` | 暗色/亮色一键切换 |
| 右键菜单 | **@radix-ui/react-context-menu** | Radix 原生，shadcn/ui 有封装 |
| Toast / 通知 | **sonner** | 轻量通知库 |

### 3.3 路由

| 类别 | 选型 | 说明 |
|------|------|------|
| 路由 | **TanStack Router** | 类型安全路由（路径参数/search params 自动推导类型）、文件系统路由可选、支持嵌套布局 |

```tsx
// TanStack Router 类型安全示例
const fileRoute = createFileRoute('/files/$path')({
  component: FileListPage,
  validateSearch: (search) =>
    z.object({ page: z.number().default(1), perPage: z.number().default(200) }).parse(search),
});

// 组件中直接使用，类型自动推导
function FileListPage() {
  const { path } = fileRoute.useParams();         // string
  const { page, perPage } = fileRoute.useSearch(); // number
}
```

### 3.4 动画

| 类别 | 选型 | 说明 |
|------|------|------|
| 页面过渡/共享布局 | **motion** (framer-motion v11) | `initial/animate/exit` + `AnimatePresence` + `LayoutGroup` |
| 列表增删动画 | motion | `layout` prop + `exit` 变体 |
| 拖拽排序 | **@dnd-kit/core** + **@dnd-kit/sortable** | 文件拖拽移动、目录排序 |
| 数值动画 | motion `useMotionValue` + `animate()` | 上传进度百分比 |

```tsx
// 路由切换动画
<AnimatePresence mode="wait">
  <motion.div key={routeId}
    initial={{ opacity: 0, x: 20 }}
    animate={{ opacity: 1, x: 0 }}
    exit={{ opacity: 0, x: -20 }}
    transition={{ duration: 0.15 }}
  >
    <Outlet />
  </motion.div>
</AnimatePresence>

// 文件列表删除动画
<motion.tr
  layout
  exit={{ opacity: 0, height: 0, marginBottom: 0 }}
  transition={{ duration: 0.25 }}
>
  ...
</motion.tr>
```

### 3.5 状态管理与请求

### 3.6 表单

| 类别 | 选型 | 说明 |
|------|------|------|
| 表单状态 | **React Hook Form** | 非受控模式、高性能（不触发全量重渲染） |
| 校验 | **Zod** | 类型推导 + runtime 校验合一 |
| shadcn 集成 | `@hookform/resolvers` | 一行 `zodResolver(schema)` |

```tsx
const createStorageSchema = z.object({
  mountPath: z.string().min(1).startsWith('/'),
  driver: z.enum(['Local', 'S3', 'FTP', 'WebDAV', 'SFTP']),
  cacheExpiration: z.number().min(0).max(1440),
});

function CreateStorageForm() {
  const form = useForm<z.infer<typeof createStorageSchema>>({
    resolver: zodResolver(createStorageSchema),
  });
  ...
}
```

## 四、前后端通信

### 4.1 通信方式

| 场景 | 方式 | 说明 |
|------|------|------|
| 管理 API | HTTP REST + JSON | `/api/admin/*`、`/api/fs/*` |
| 文件下载 | 302 重定向 或 流式代理 | `/d/*path`、`/p/*path` |
| 文件上传 | `PUT` 流式 或 `multipart/form-data` | `/api/fs/put`、`/api/fs/form` |
| WebDAV | HTTP WebDAV 协议 | `/dav`，Windows/macOS 挂载 |
| 实时通知 | WebSocket | `/api/admin/message` |
| AI 集成 | MCP 协议 (STDIO/HTTP) | `server/mcp/` |

### 4.2 认证方式

| 方式 | 使用场景 |
|------|----------|
| JWT Token (`Authorization: Bearer <token>`) | Web 前端登录后请求 |
| Admin Raw Token (`Authorization: <raw_token>`) | 管理员超权操作 / 脚本自动化 |
| Basic Auth | WebDAV 挂载、Windows/macOS 连接 |
| SSO OIDC | 企业统一登录 |
| LDAP | 企业目录认证 |
| WebAuthn | 无密码生物识别 |
| TOTP 2FA | 登录二次验证 |

### 4.3 下载签名

所有 `/d/*path` 和 `/p/*path` 路由需携带 `?sign=<hmac>` 参数，签名由后端 HMAC-SHA256 计算：

```ts
// 前端生成签名 URL
const sign = await api.post('/api/fs/get', { path }).then(r => r.data.data.sign);
const url = `/d${path}?sign=${sign}`;
```

## 五、开发工作流

### 5.1 日常开发

```bash
# 终端 1：Go 后端
go run . server --dev

# 终端 2：前端 Vite Dev Server
cd web && pnpm dev
# Vite 自动 proxy /api → localhost:5244
# 访问 http://localhost:5173
```

### 5.2 生产构建

```bash
# 先构建前端
cd web && pnpm build         # → public/dist/

# 再编译 Go（内嵌前端）
CGO_ENABLED=0 go build -ldflags="-w -s" -o alist .
# 产物：alist（单文件，含全部前端资源）
```

## 六、技术栈选型一览

| 层 | 选型 | 关键决策理由 |
|----|------|-------------|
| 后端语言 | Go 1.25 | 原生并发、单文件部署、低内存 |
| HTTP 框架 | Gin | 生态最成熟、中间件丰富 |
| ORM | GORM | AutoMigrate 省建表、Hook 自动 JSON 序列化 |
| 数据库 | MySQL / PostgreSQL | 生产环境，MySQL 默认 |
| 前端框架 | React 19 | 生态最广、AI 辅助最强、shadcn/ui 绑定 React |
| 组件库 | shadcn/ui | 代码在本地、可改源码、不依赖 npm 包生命周期 |
| 动画 | motion (framer-motion) | 声明式 API、AnimatePresence 原生支持、生态最成熟 |
| 路由 | TanStack Router | 类型安全 search params、无需手写类型守卫 |
| 请求 | TanStack Query + Axios | 自动缓存/分页/重取、拦截器统一 Token 刷新 |
| 状态 | Zustand | 轻量、无 boilerplate |
| 表单 | React Hook Form + Zod | 非受控高性能 + 类型推导校验 |
| 虚拟滚动 | react-virtuoso | 10 万行不卡、不等高行原生支持 |
| 构建 | Vite + pnpm | 启动 < 500ms、严格依赖树 |
| 代码规范 | @antfu/eslint-config | 开箱即用、零配置 |
