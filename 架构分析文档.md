# 项目架构全览 —— 分层分析

## 0. 整体结论

项目采用**前后端分离**架构，前端为独立 SolidJS 项目（alist-web），后端为 Go/Gin RESTful API。**无 tRPC、无 Server Action**，API 通信全为 HTTP REST。后端分层清晰：Router → Middleware → Handler → fs 层 → op 层 → Driver → DB。

## 2. 分层详解表

| 层 | 目录/文件 | 技术栈 | 职责 | 通信方式 | 关键设计 |
|----|-----------|--------|------|----------|----------|
| **前端** | `public/dist/` (独立仓库 alist-web) | SolidJS + Vite + Axios | 网页门户 UI，文件浏览/管理/预览 | HTTP REST → 后端 API | SPA 模式，由 `server/static/static.go` 注入 `index.html` 配置；**无 tRPC，无 Server Action** |
| **路由** | `server/router.go` | Gin | 注册全部 API 路由和中间件栈 | — | 按功能分组：`/api/fs/*` 文件操作，`/api/admin/*` 管理，`/dav` WebDAV，`/s3` S3；签名下载路由 `/d/*path`、`/p/*path` |
| **中间件** | `server/middlewares/` | Gin Middleware | 认证/授权/限速/签名/并发控制 | ctx.Get/Set 传递 user/meta | `Auth`: JWT→Token→Guest 三级 fallback；`UploadRateLimiter`/`DownloadRateLimiter`: 包裹 io.Reader/Writer；`MaxAllowed`: 信号量并发控制 |
| **Handler** | `server/handles/` (30 文件) | Gin Handler | 请求校验 → 调用 fs 层 → 响应序列化 | 通过 `fs.*` 调用 | 每个 handler 模式: 绑定参数 → 权限检查 → `fs.Xxx()` → `common.SuccessResp()`；密码保护 meta 检查；分页/过滤/排序 |
| **FS 层** | `internal/fs/` (10 文件) | Go | 挂载路径→存储驱动+实际路径 的转换 | 调用 `op.GetStorageAndActualPath()` | 所有函数的 `path` 参数是挂载路径；List 支持虚拟文件(挂载点目录)和隐藏规则过滤；Copy 支持跨存储复制(下载→上传管道) |
| **OP 层** | `internal/op/` | Go | 实际操作编排：调用驱动 + 缓存 + 去重 + 排序 | 调用 `storageDriver.List/Link/Put...` | `listCache`(内存60s) + `linkCache`(按Link过期) + `singleflight` 防击穿；`storagesMap` 内存存储注册表，最长路径匹配 + Round-Robin 负载均衡 |
| **驱动接口** | `internal/driver/driver.go` | Go Interface | 定义统一存储抽象(Duck Typing) | — | `Driver = Meta + Reader` 是必须的，Writer/ArchiveReader 等通过类型断言(`ok pattern`)可选实现；`Link` 返回 URL/MFile/RangeReadCloser 三种模式 |
| **驱动注册** | `drivers/all.go` + `internal/op/driver.go` | Go Reflect + init() | 驱动自动发现与注册 | `import _` 副作用 + `init()` 调用 `op.RegisterDriver()` | 使用 `reflect` 解析每个驱动的 Addition 结构体 tag，生成前端表单元数据；驱动名作为唯一 Key 存入 `driverMap` |
| **本地驱动** | `drivers/local/` | os + otaiti10/copy + ffmpeg | 本地文件系统完整 CRUD | 直接调用 OS API | 支持缩略图(图片/视频/ffmpeg)、回收站、跨设备 Move fallback |
| **协议客户端驱动** | `drivers/ftp/` `drivers/webdav/` `drivers/sftp/` `drivers/smb/` | jlaffaye/ftp + gowebdav + pkg/sftp + go-smb2 | 远程协议客户端挂载 | FTP/WEBDAV/SFTP/SMB 协议 | 各自封装为统一 Driver 接口，对外透明 |
| **对象存储驱动** | `drivers/s3/` `drivers/azure_blob/` `drivers/upyun/` | AWS SDK / Azure SDK / UPYUN SDK | 对象存储挂载 | S3/HTTPS 协议 | 完整 List/Link/Upload/Copy/Remove |
| **协议服务端** | `server/webdav.go` `server/ftp.go` `server/sftp.go` `server/s3.go` `server/mcp/` | net/http + ftpserverlib + sftpd + gofakes3 + mcp-go | 对外暴露多协议文件服务 | WebDAV/FTP/SFTP/S3/MCP | 各自独立端口(localhost:5221/5222/5246/5248)，共用统一的 Auth 用户认证和文件系统 |
| **MCP 服务** | `server/mcp/` `cmd/mcp.go` | mcp-go v0.48.0 | AI 助手集成 (Model Context Protocol) | HTTP Streamable + STDIO 双模式 | 提供 `tools_read.go`(浏览/搜索/下载)、`tools_manage.go`(CRUD)、`tools_upload.go`；支持用户级权限 |
| **ORM** | `internal/db/` (15 文件) | GORM v1.25.11 | 数据库持久化 | SQLite3 (默认) / MySQL / PostgreSQL | 13 张表 AutoMigrate；User.Role 字段为 JSON 数组用 `JSON_CONTAINS` 查询；Role 的 GORM 钩子自动序列化/反序列化 PermissionScopes → RawPermission |
| **Setting** | `internal/setting/` + `internal/op/setting.go` | Key-Value + 回调 | 全局配置读写(内存缓存 + DB) | `settingItemCache` + DB SettingItem 表 | 支持变更回调：限速器、索引等动态更新无需重启；16 个分组(SITE/PREVIEW/TRAFFIC/SSO 等) |
| **Stream/限速** | `internal/stream/` | `golang.org/x/time/rate` Token Bucket | 流式传输限速 | 包裹 io.Reader/Writer/File/RangeReadCloser | 4 个限速器常量；`blockBurstLimiter` 解决大 burst buffer 问题；支持 Context 取消检测 |
| **流式传输** | `internal/stream/stream.go` | io.Pipe | SeekableStream: 内存/RAM Disk/TempFile 三级 | — | 自动判断: ≤4MB 内存 → ≤RAM disk → TempFile；支持 `ReadAt`+`RangeRead` |
| **归档** | `internal/archive/` + `drivers/base/archive.go` | mholt/archives + bodgit/sevenzip | zip/rar/7z/tar 等格式浏览和解压 | — | 支持密码、内部文件列表、流式提取；驱动可自定义解压逻辑 |
| **离线下载** | `internal/offline_download/` | ARIA2/qBittorrent/Transmission RPC | 离线下载管理 | ARIA2 JSON-RPC / qBT HTTP API | 统一接口，自动选工具；`pkg/aria2/` 和 `pkg/qbittorrent/` 封装 RPC 客户端 |
| **搜索** | `internal/search/` | Bleve / Meilisearch | 全文搜索 | Bleve 本地索引 / Meilisearch HTTP | 支持增量索引、进度查询、路径排除、深度限制 |
| **FRP** | `internal/frp/` | fatedier/frp v0.68.0 | 内网穿透 (Fast Reverse Proxy) | FRP 协议 | 自动注册到 FRP 服务器，支持自定义域名/子域名/远程端口 |
| **认证** | `internal/authn/` `internal/session/` `internal/device/` | JWT + WebAuthn + OIDC + LDAP | 多方式认证与会话管理 | — | JWT 48h 过期 + PwdTS 防旧Token；设备级会话(IP+UA)；Session 支持 WebSocket 通知驱逐 |
| **配置** | `internal/conf/` `internal/bootstrap/` | cobra + env + JSON config | 启动初始化与全局配置 | `config.json` + 环境变量 | `caarlos0/env` 自动解析环境变量前缀；DefaultConfig() 定义全默认值；bootstrap 按序 Init |
| **WebSocket** | `internal/message/` | gorilla/websocket | 管理员消息推送 | WebSocket (仅此一处) | `/api/admin/message` 用于实时通知 |

## 4. 关键设计模式

| 模式 | 应用位置 | 说明 |
|------|----------|------|
| **接口 + 类型断言** | `driver.Driver` + 所有 `*Result` 接口 | 驱动能力可选实现，运行时通过 `storage.(driver.MoveResult)` 判断 |
| **Duck Typing** | `driver.Driver` 不是固定结构，而是小接口组合 | 新驱动只需实现 Meta + Reader，写入按需加 MakeDir/Put/Copy 等 |
| **注册表模式** | `op.driverMap` + `op.storagesMap` | 驱动工厂函数 + 存储实例 都是全局 Map，`import _` 副作用注册 |
| **装饰器模式** | `stream.RateLimitReader/Writer/File/RangeReadCloser` | 包裹原始 IO 对象增加限速能力 |
| **单飞 (Singleflight)** | `op.listG` / `op.linkG` / `op.mkdirG` | 同一 Key 并发请求合并为一次实际调用，防缓存击穿 |
| **缓存 + Debounce** | `op.listCache` / `op.addSortDebounceMap` | 目录列表缓存 60s，新增文件后延迟排序 |
| **中间件链** | Gin Middleware | Auth → Sign → RateLimit → Handler，逐层注入 context |
| **策略模式** | `proxy.go` 三种下载模式 | 根据 `link.MFile` / `link.RangeReadCloser` / `link.URL` 选择传输策略 |
| **三级限速** | `stream/` 四个 Limiter | ClientDownload / ClientUpload / ServerDownload / ServerUpload |
| **最长路径匹配** | `op.getStoragesByPath()` | 多级挂载路径按 `/` 数量精确匹配，同深度支持负载均衡 |
| **负载均衡** | `op.GetBalancedStorage()` | 多个存储挂载到同一路径时 Round-Robin 分配 |
| **Hook 机制** | `op.callStorageHooks()` | 存储增删改时触发回调 |

## 6. 前端 API 调用方式

```
前端 (SolidJS/Axios)
  │
  ├─ POST /api/auth/login          → JWT Token
  ├─ POST /api/fs/list             → 文件列表 (带分页/路径/密码)
  ├─ POST /api/fs/get              → 文件详情 (含 raw_url)
  ├─ POST /api/fs/mkdir            → 新建目录
  ├─ POST /api/fs/rename           → 重命名
  ├─ POST /api/fs/move             → 移动
  ├─ POST /api/fs/copy             → 复制
  ├─ POST /api/fs/remove           → 删除
  ├─ PUT  /api/fs/form             → 表单上传
  ├─ PUT  /api/fs/put              → 流式上传
  ├─ GET  /d/*path                 → 直接下载 (302重定向或本地流)
  ├─ GET  /p/*path                 → 代理下载 (透明转发)
  ├─ GET  /api/fs/search           → 全文搜索
  │
  ├─ 所有管理接口 → /api/admin/*     → 需 admin 角色
  └─ WebSocket → /api/admin/message → 消息通知 (仅此一处非 REST)

认证方式:
  1. JWT (Authorization: Bearer <token>) — 默认
  2. Admin Token (Authorization: <raw_token>) — 管理超权
  3. SSO/OIDC — 第三方登录
  4. LDAP — 企业目录登录
  5. WebAuthn — 无密码生物识别
  6. WebDAV Basic Auth — Windows/macOS 挂载用
```
