## 二、全 Mock 组件 — 有 UI 但无真实 API 调用

以下 6 个功能模块 UI 完整，但所有操作仅执行 `console.log` + `toast.success()`，无任何 HTTP 请求。

| # | 组件 | 文件 | 应调用的后端 API | 工作量 |
|---|------|------|-----------------|--------|
| 1 | 传输任务管理 | `file/TransferDrawer.tsx` | `/api/admin/task/upload|copy|offline_download/*` | 中 |
| 2 | 全文搜索 | `search-dialog.tsx` | `POST /api/fs/search` | 小 |
| 3 | WebDAV 服务配置 | `admin/WebDAVSettings.tsx` | `/api/admin/setting/get|save` | 小 |
| 4 | HTTPS 证书配置 | `admin/HttpsSettings.tsx` | `/api/admin/setting/get|save` | 小 |
| 5 | 传输限速配置 | `admin/TransferLimitSettings.tsx` | `/api/admin/setting/get|save` | 小 |
| 6 | 协议端口管理 | `admin/ProtocolPortSettings.tsx` | `/api/admin/setting/get|save` | 小 |

### 详情

**传输任务管理** — 传输列表/进度/暂停/继续/清除 全部 `logMockAction()`。`TransferBar` 底部状态条使用 `mockTransfers` 硬编码数据。

**全文搜索** — 输入框无 `onChange` 处理，最近搜索来自 `mockRecentSearches` 假数据，点击搜索项无任何操作。

**4 个管理设置面板** — WebDAV/HTTPS/限速/端口 的开关和保存按钮均只打 `logMockAction()` 日志，无需对接后端。后端 `/api/admin/setting/*` 全部就绪。
